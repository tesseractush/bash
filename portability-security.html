Portability & Security

This module covers best practices for writing portable and secure Bash scripts.

Shebang portability (#!/usr/bin/env bash)

Description: Use #!/usr/bin/env bash for scripts to work on different systems.

Syntax: #!/usr/bin/env bash

Example: #!/usr/bin/env bash\necho "Portable script"

Tip: Avoid hardcoding interpreter paths.

Input sanitization

Description: Always validate and sanitize user input to prevent security issues.

Syntax: if [[ $input =~ ^[a-zA-Z0-9_]+$ ]]; then ...

Example: read name; if [[ $name =~ ^[a-zA-Z]+$ ]]; then echo "Valid"; fi

Tip: Never use unsanitized input in commands or file paths.

Restricted shells

Description: Limit user capabilities by using restricted shells (e.g., rbash).

Syntax: rbash

Example: ln -s /bin/rbash /bin/sh_restricted

Tip: Use for guest or limited accounts to enhance security.
