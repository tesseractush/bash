Arguments & Exit Codes

This module covers how to handle script arguments and exit codes in Bash.

$1, $2, $@, $#

Description: Special variables for accessing script arguments.

Syntax: $1 $2 ... $@ $#

Example: echo "First arg: $1, All args: $@, Arg count: $#"

Tip: $@ expands to all arguments, $# gives the count.

$?

Description: The exit status of the last command (0 = success, nonzero = error).

Syntax: echo $?

Example: ls /notfound; echo $? # prints nonzero

Tip: Always check $? after critical commands.

Custom exit codes

Description: Use exit [code] to set a script's exit status.

Syntax: exit 0 (success), exit 1 (error)

Example: if [ ! -f file ]; then echo "Missing file"; exit 2; fi

Tip: Use unique codes for different error types.
